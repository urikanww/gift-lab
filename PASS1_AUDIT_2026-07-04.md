## 3. B2B readiness verdict

**NOT READY.** The order spine itself is genuinely strong — both production gates are enforced server-side and survived adversarial runtime testing, proof approval is immutable, pricing is consistent to the cent across designer/cart/quote/PO, the licence gate blocks every non-commercial variant, and the full journey ran end-to-end today. But a real first-time corporate buyer cannot get through it: there is no account registration (the flow dead-ends at the login wall), the environment shipped with an unapplied migration that 500'd quote creation until fixed mid-audit, the designer silently swallows invalid/oversized/low-res uploads on the exact step the business flagged as its known problem area, name personalization is missing entirely, and staff cannot adjust a single price without database access. These are launch blockers, not polish items.

## 4. Top 5 customization friction points + fixes

1. **Silent upload failures (C1/C2/C3).** txt/PDF ignored without a word; SVG sneaks into canvas via drag-drop then dies server-side later; 36 MB and 40×40 px files accepted mutely. *Fix:* one client-side validator on file select/drop — type allowlist, size cap, min-resolution vs selected band — each failure raising a specific toast ("PDF isn't supported — export as PNG or JPG").
2. **Logo can leave the print area or the canvas entirely (C7).** Clamp exists only at add time; a dragged logo can sit half outside the dashed zone, or vanish off-canvas while "1 element" still shows — and the out-of-zone result is what gets printed (C9). *Fix:* clamp `object:moving`/`object:modified` to the PRINT_INSET rect (same pattern as the existing band clamp) and block "Use this design" while any object is out of zone.
3. **Price jumps between designer and cart (SGD 25 setup fee).** Designer footer says 2120.50, cart says 2145.50 with no line explaining why. *Fix:* show "+ SGD 25.00 artwork setup" in the designer estimate breakdown, or fold it into the displayed total from the first screen.
4. **Login redirect breaks its promise.** Checkout says "you'll come right back here to finish", then login lands on My Orders; buyer must re-find checkout. *Fix:* honor a `?redirect=/checkout` param on login success.
5. **Two-step save (Use this design → Add to cart) with stale hints.** The sticky bar keeps saying "Add a logo, then choose…" even after a logo exists, and nothing forces re-saving after an edit (stale artwork risk — canvas edits after save aren't re-captured unless the user re-saves). *Fix:* single primary action that captures-then-adds, hint bound to actual canvas state, and dirty-state detection that invalidates the previous capture after any edit.

## 5. Spec gaps requiring business decisions (not dev fixes)

1. **D8 (standing item):** logo-size mechanism (ratify user-selected band), band boundaries (14–26/26–40/40–56 % of canvas), L-tier existence and price, and reconciliation of spec's $3/$5 per-logo fee with the implemented flat-8 + setup-25 + per-unit 0/0.40/0.90 structure. Update both spec docs.
2. **Buyer onboarding model:** self-serve registration at Request Quote (spec Stage 0) vs v1 "provisioned by account manager" (implemented). If provisioning is the launch model, spec and sales process must say so; if not, registration is a build item.
3. **Reconfirmation actor:** spec says the *client* chooses amend/drop/cancel on qty-short/price-jump; implementation makes it staff-recorded (buyer gets 403). Decide client-facing vs ops-mediated for v1.
4. **Invoice artifact:** reference-only (implemented) vs generated invoice document (spec §Stage 4 "system generates an invoice"). Decide format + numbering + tax fields.
5. **"Pin" curation (spec 6.7):** wanted for launch or dropped?
6. **Multi-logo pricing:** second logo currently free (fee keyed to one logo_size). Charge per placement or cap at one logo per item?

## 6. Ordered fix list (blockers first, each with acceptance criterion)

1. **Deploy-drift guard: migration `2026_07_04_000025_add_needed_by_to_quotes` was unapplied in the running environment; every quote creation 500'd** (fixed during audit by `php artisan migrate`). *Accept:* CI/deploy runs `migrate --force` before serving traffic, plus a smoke test that POSTs a quote and expects 201.
2. **Buyer registration (A2)** — or formally adopt provisioned-only onboarding (then update spec + login copy + sales SOP). *Accept:* a brand-new company can browse → design → request quote unassisted; or spec v1.1 states provisioning and audit check A2 is rewritten.
3. **D8 decision packet** (mechanism, boundaries, L price, fee structure) signed off and written into both spec docs. *Accept:* next audit run returns D8 = PASS.
4. **Upload validation UX (C1/C2/C3)** — type/size/resolution checks with specific errors, both on file-picker and drag-drop paths. *Accept:* txt, pdf, svg, 36 MB png, 40×40 png each produce a distinct visible error; png/jpg within limits still work.
5. **Print-area enforcement (C7/C9)** — move/modify clamp + out-of-zone save block. *Accept:* logo cannot be dropped outside the dashed zone; "Use this design" disabled (with reason) if any element out of zone; off-canvas loss impossible.
6. **Superadmin pricing editor (E1/D7/E2)** — CRUD over `pricing_configs` (superadmin-gated, audited) incl. margin floor, size fees, setup fee, drift %, pay-now cutoff. *Accept:* superadmin changes M surcharge in UI; next designer estimate reflects it with no deploy/DB access.
7. **Name/text personalization (D9/D10)** — second designer mode, combinable with logo, priced via existing `customization_per_unit`. *Accept:* item with logo+name prices flat + logo surcharge + per-unit text fee additively; FA contains the text layer.
8. **Invoice document (A7)** — generate numbered invoice (PDF or hosted) from PO data. *Accept:* PO issuance produces a retrievable invoice artifact attached to the quote.
9. **FA metadata persistence (C12)** — store layout JSON (coords, scale, band, canvas-to-product mm mapping) server-side with the artwork ref. *Accept:* production job exposes resolution, physical placement, size, method without opening the PNG.
10. **CORE product/variant admin (E4)** — minimal create/edit/stock UI. *Accept:* ops adds a new blank with variants + stock without touching seeders. (Interim guard: block quoting variant-less CORE products — today Alpha Mug was orderable but structurally unprocurable, runtime-proven.)
11. **Checkout polish (friction 3/4/5)** — setup-fee disclosure, login redirect return, single-action save+add with dirty-state. *Accept:* walkthrough shows same total designer→cart, post-login return to checkout, edited canvas cannot be submitted stale.
12. **Rotate control (C6)** — enable Fabric rotation handle + angle in layout. *Accept:* logo rotatable on desktop + mobile; angle survives into FA.
13. **Code-side tech-debt marker for scraped images (B8)** + "pin" decision (spec gap 5). *Accept:* tracked issue linked in `ScrapedCatalogueService`; pin built or de-scoped in spec.
